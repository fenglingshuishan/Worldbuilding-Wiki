# 技术架构

## 1. 总体结构

```text
本机浏览器
    │ HTTP（仅 127.0.0.1）
    ▼
Web 应用
    ├── 条目与模板服务
    ├── 链接、关系与时间服务
    ├── 搜索与一致性检查
    └── 导入、导出与备份
        │
        ├── vault/ Markdown + YAML + 附件（真源）
        └── runtime/index.sqlite（可重建派生数据）
```

首版建议采用 Python Web 应用、服务端模板配合少量渐进式交互、SQLite FTS 全文检索。这样能复用 Python 的 Markdown/YAML 生态，减少个人工具的构建链和前后端重复模型。技术选型进入编码时再用小型原型验证，不在设计文档中锁死具体库版本。

## 2. 模块边界

| 模块 | 职责 | 禁止承担 |
| --- | --- | --- |
| Vault | 安全读写 Markdown、YAML 和附件 | 搜索排序、AI 推断 |
| Parser | 解析元数据、链接、嵌入和关系 | 修改文件 |
| Indexer | 构建全文索引、反向链接和派生表 | 成为唯一数据源 |
| Domain | 校验实体、时间、分支和关系规则 | 处理 HTTP 或页面渲染 |
| Checker | 产出带证据的冲突候选 | 自动修正正史 |
| Web | 路由、表单、编辑器和视图 | 绕过 Domain 直接写文件 |
| AI Adapter | 检索上下文、调用模型、生成差异提案 | 直接写入正文 |

## 3. 写入流程

1. Web 层接收编辑内容和当前文件版本。
2. Domain 层解析并校验字段、链接和关系。
3. Vault 层检查乐观锁，写入同目录临时文件并原子替换。
4. Indexer 在单个事务内更新该条目相关索引。
5. Checker 增量计算受影响的检查结果。
6. 如索引失败，返回“正文已保存、等待重建索引”，而不是回滚或破坏正文。

每次写入保留内容哈希，防止浏览器中的旧页面覆盖磁盘上由其他编辑器更新的版本。

## 4. SQLite 的边界

SQLite 保存：

- 条目标题、类型、状态和文件定位；
- 全文检索数据；
- 解析后的链接、反向链接和关系；
- 时间范围的规范化表示；
- 检查结果和任务状态；
- 可丢弃的缩略图与计算缓存。

SQLite 不保存正文的唯一副本，不保存无法从文件恢复的关系，也不保存密钥。提供 `reindex` 命令删除并完整重建索引。

## 5. 搜索策略

中文搜索至少需要同时处理完整词、连续字符和别名。MVP 先验证 SQLite FTS 的实际中文召回率；若不足，再增加字符 n-gram 派生字段。搜索排序综合标题精确匹配、别名匹配、正文相关度、条目状态和最近编辑时间，但不能让“最近编辑”淹没正确标题。

## 6. 一致性检查架构

每条检查规则包含稳定规则 ID、严重级别、依赖数据、证据列表和修复建议。例如：

- `LINK_BROKEN`：链接无法解析；
- `ALIAS_AMBIGUOUS`：同一别名对应多个条目；
- `DEPRECATED_REFERENCED`：正史引用废弃条目；
- `RELATION_OVERLAP`：互斥任职期确定重叠；
- `EVENT_ORDER_INVALID`：结果事件确定早于原因事件；
- `LOCATION_CONFLICT`：同一人物在不可兼容时间范围出现于多个地点。

检查结果可以被“确认无误”“暂时忽略”或“已解决”，但忽略记录要绑定规则、证据哈希和原因；相关内容变化后重新检查。

## 7. 安全与隐私

- 默认仅绑定 `127.0.0.1`，不对局域网或公网开放。
- Markdown 渲染过滤脚本、危险 URL 和原始 HTML。
- 文件路径规范化后必须位于 `vault/` 内，防止目录穿越。
- API 密钥只从环境变量或本机密钥存储读取，不写入 vault 或日志。
- AI 调用前展示将发送的条目范围；默认关闭外部 AI。
- 若以后允许局域网访问，必须增加身份验证、CSRF 防护和 TLS 终止方案。

## 8. 可靠性与备份

- 文件写入使用临时文件、刷新和原子替换。
- 自动备份是 vault 快照，不是 SQLite 复制。
- 支持手动 Git 检查点；默认不在每次按键时产生提交。
- 启动时检测未完成临时文件、外部修改和索引版本。
- 数据格式升级先备份，再执行可回滚迁移。
- 导出功能必须在不运行本应用时仍可阅读。

## 9. 部署边界

个人使用阶段提供一个本机启动命令和清晰的数据目录配置。以后可以打包为桌面启动器或容器，但容器不能成为访问本地文件的必要条件。公网发布应作为独立只读导出器设计，不能直接暴露作者工作库。
