# 用户指南

- 适用版本：0.4.0
- 受众：使用独立发行包的个人创作者
- 数据影响：所有可编辑世界观内容都写入用户选择的世界库

## 1. 启动与停止

解压与操作系统匹配的发行包。Windows 双击 `WorldbuildingWiki.exe`；Linux 在发行目录运行：

```bash
./WorldbuildingWiki
```

程序启动本机服务并打开浏览器。地址默认是 `http://127.0.0.1:3764`；若端口被占用，命令行可指定其他端口。浏览器标签页只是界面，关闭它不会停止程序。停止时进入“设置”，选择“退出本地程序”；终端运行也可按 `Ctrl+C`。

程序默认不向局域网或公网开放。

## 2. 第一次运行

可选择三种方式：

1. **新建世界库**：填写世界库和第一个世界的名称；默认同时载入“示例全套数据”，也可取消勾选。
2. **打开现有目录**：选择或填写包含 `vault.yaml` 的目录。
3. **导入传输包**：选择 `.worldvault`，验证后创建一个新的世界库。

一个世界库可以包含多个世界。在“设置”中使用“创建另一个世界”。世界库与程序安装目录相互独立，替换程序不会删除正文。

### 示例全套数据

“潮汐档案 · 示例全套数据”是一套随程序发布的虚构演示世界，包含 13 条概念、规则、地点、人物、组织、事件、文化、物件与来源内容，以及关系、时间、自定义字段模板和可交互地图。示例条目均带有固定的 `sample_set` 标记和“示例全套数据”标签。

首页“示例全套数据”卡片提供三个操作：

- **打开示例**：从世界概览开始浏览实际效果；
- **删除**：仅删除固定 ID 且带正确示例标记的条目、示例模板和示例地图；
- **还原**：重新写入标准示例，并把改过的示例条目恢复为初始内容。

如果你在示例世界内新建了自己的条目，删除示例时该条目和世界目录都会保留。还原也不会覆盖没有示例标记的内容；固定 ID 被非示例内容占用时，应用会拒绝操作并报告冲突。

### 关闭与删除世界库

“关闭世界库”只让应用断开当前目录，不删除任何文件，之后仍可从近期记录或原目录重新打开。

需要永久删除时，进入“平台设置”的“危险操作”，选择“永久删除当前世界库”，并准确输入界面显示的世界库名称。删除会移除该世界库中的 Markdown 正文、YAML、地图和附件，同时清理对应的 SQLite 索引与近期记录。该操作不可撤销，也不会生成隐藏备份；如果内容可能继续使用，请先在“迁移与发布”中导出 `.worldvault`。

## 3. 创建和编辑条目

点击“新建条目”，选择类型和状态。必填项只有标题；建议先写一段最确定的设定，再补充结构化字段。

也可以从“模板中心”选择内置创作框架，或建立自己的模板。自定义模板保存在当前世界库中，会随完整世界传输包一起迁移；模板只影响创建时的初始内容，不会持续绑定条目。

自定义模板可定义文本、数字、是/否、单选和列表字段。升级模板会递增版本并保留旧 YAML，已创建条目继续引用原版本；删除模板也保留被条目引用的旧 schema。“迁移条目”允许选择最多 200 条内容，把来源字段 ID 映射到目标字段；必须先预览必填字段与默认值风险，通过后才原子应用。

### 查看差异与恢复误改

打开条目后选择“版本历史”。应用在每次应用内保存、批量修改或标记废弃之前自动保存旧版本，并为每条条目保留最近 100 个快照。历史页用绿色和红色分别显示相对当前内容的新增与删除行。

选择历史版本并点击“恢复这个版本”会恢复标题、状态、分支、标签、关系、时间信息和正文。恢复请求仍检查当前内容哈希；若其他页面或外部编辑器已经修改文件，应用会拒绝覆盖并要求重新加载。恢复前也会保存当前版本，因此可以通过历史页撤回一次错误恢复。

历史快照位于世界库的隐藏 `.history/entries/`。直接在外部编辑器中发生的修改无法追溯其修改前内容，但应用下一次写入前会保存它看到的当前外部版本。

## 4. 管理创作进度

“总览”汇总条目规模、正史比例、草稿和知识连接，并提供最近编辑、示例管理与常用入口。“条目”支持按世界、类型、状态和关键词筛选；勾选多个条目后，可以批量调整内容状态或追加标签。单次批量操作上限为 200 条。

“关系网络”同时显示编辑器中建立的语义关系和正文中的 Wiki 提及。语义关系表示明确事实，正文提及只表示内容之间存在引用，两者不会互相冒充。

内容状态：

| 状态 | 用途 |
| --- | --- |
| 正史 | 作者确认的当前事实，参与全部一致性检查 |
| 草稿 | 尚未采用的候选设定 |
| 传闻 | 世界内角色或组织的说法 |
| 废弃 | 保留历史但不再有效的设定 |

正文支持 Markdown。使用 `[[雾港]]` 链接标题或别名，使用 `[[loc_abcd1234abcd|雾港]]` 锁定稳定 ID。修改标题不会破坏 ID 链接。

“关系”用于强语义，例如：

- `member_of`：属于某组织；
- `located_in`：位于某地点；
- `causes`：导致某事件；
- `precedes`：发生在某事件之前；
- `parent_of`、`ally_of`、`enemy_of` 等。

关系目标填写目标条目的稳定 ID。普通提及不要伪装成强关系，直接写 Wiki 链接即可。

## 5. 时间与分支

“时间显示”保存世界内说法，例如“星历 312 年霜月下旬”；最早和最晚序号用于排序与模糊范围。如果只有大约时间，可填写一个范围，不要伪造精确日期。

时间线分支默认是 `main`。在“时间线与分支”可从现有条目建立谱系变体。比较页区分继承、新增、覆盖、已同步和重复变体冲突；每项可选择保留基准、采用目标或复制为基准分支草稿。采用目标前会保存基准条目历史，不会隐式批量覆盖。

## 6. 图片、地图和附件

编辑条目时选择附件并点击“上传并插入正文”。应用接受常用图片、PDF、音频、TXT 和 CSV，单文件不能超过 50 MiB。附件保存在当前世界的 `assets/`，通过相对 Markdown 路径引用，并随 `.worldvault` 一起迁移。

“地图工作区”接受 PNG、JPEG、WebP 和 GIF 原图，单张上限同为 50 MiB。添加地图后可建立最多 50 个图层、切换图层可见性，并从同一世界的地点条目中选择目标后点击地图放置标记。点击已有标记可以移除它；删除标记不会删除地点条目。

地图原图与同目录的 `<map-id>.yaml` 分开保存。YAML 只记录图层、地点稳定 ID 和 0–1 范围内的横纵坐标，所以缩放界面不会改变地点位置。每张地图最多保存 1000 个标记。删除地图会同时删除其原图和 YAML，但不影响任何地点条目。

不接受 HTML、脚本和可执行文件。Markdown 中的原始 HTML不会执行。

## 7. 可选 AI 提案

“AI 提案”默认关闭。启用本地模型时设置 `WORLDBUILDING_AI_MODE=local`、`WORLDBUILDING_AI_MODEL`，可用 `WORLDBUILDING_AI_ENDPOINT` 覆盖默认回环端点；本地模式拒绝非回环地址。外部兼容端点使用 `external` 模式，必须是 HTTPS，并可通过 `WORLDBUILDING_AI_API_KEY` 提供密钥。

一次最多选择 20 个条目、合计提示最多 60000 字。必须先预览条目、字段、正文长度和端点，范围或任务文字变化后需重新确认。模型输出只写入应用运行目录的提案区，不会修改 Markdown；删除提案也不影响世界库。

## 8. 搜索、时间线与检查

顶部搜索框检索标题、别名、正文和标签。“时间线”按内部序号排列有时间信息的条目。

“质量中心”目前报告：

- 断开的 Wiki 链接；
- 同名或重复别名；
- 没有关联的孤立条目；
- 正史引用废弃条目；
- 关系目标不存在；
- 原因或前序事件确定晚于结果事件；
- 同一条目的 `located_in` 时间范围重叠；
- 无法解析的 Markdown 条目文件；
- 多个文件使用同一条目 ID。

检查只提供证据，不自动修改正文。

## 9. 导出与跨设备迁移

进入“迁移与发布”。

### 世界传输包

“下载 `.worldvault`”导出可继续编辑的当前世界库快照。包中包含 Markdown、YAML、地图和附件，以及格式清单和逐文件 SHA-256；不包含 SQLite、日志、密钥、Git 元数据和隐藏的本地条目历史。若跨设备时还要保留完整编辑轨迹，请在应用关闭后备份整个世界库目录。

在另一台设备安装相同或更新版本的程序，首次运行选择“导入传输包”。向已有世界库合并时，应用分别列出：

- 新增条目；
- 内容完全相同的条目；
- 同 ID 但正文不同的条目；
- 同路径但内容不同的附件或世界配置。

冲突默认保留本地。可以采用导入版本，或把冲突条目另存为草稿副本。提交合并前自动生成恢复快照。

导入会拒绝损坏条目、重复条目 ID、校验和不符，以及在 Windows 上会发生大小写碰撞或使用保留名称的路径。验证全部在暂存区完成；失败时不会修改已有世界库。

### 静态审阅包

“下载审阅包”生成 HTML ZIP。解压后打开 `index.html`，无需安装程序即可检索和阅读。它是只读快照，不能替代 `.worldvault` 或历史备份。

## 10. 恢复与故障处理

- 索引结果不对：在“平台设置”或“质量中心”点击重建索引。
- 程序无法启动：世界库仍是普通 Markdown，可用文本编辑器读取。
- 导入失败：原世界库保持不变；根据错误重新导出或修复磁盘空间。
- 合并结果不符合预期：恢复快照位于应用运行数据的 `runtime/backups/`。
- 程序升级失败：恢复旧程序目录并继续打开同一个世界库。
- 浏览器显示服务不可用：确认本地程序仍在运行，而不是只保留了浏览器标签页。

不要把静态审阅包作为唯一备份，也不要让多台设备同时写同一网络共享世界库。
