# 用户指南

- 适用版本：0.1.0
- 受众：使用独立发行包的个人创作者
- 数据影响：所有可编辑世界观内容都写入用户选择的世界库

## 1. 启动与停止

解压与操作系统匹配的发行包。Windows 双击 `WorldbuildingWiki.exe`；Linux 在发行目录运行：

```bash
./WorldbuildingWiki
```

程序启动本机服务并打开浏览器。地址默认是 `http://127.0.0.1:3764`；若端口被占用，命令行可指定其他端口。浏览器标签页只是界面，关闭它不会停止程序。停止时进入“设置”，选择“退出本地程序”；终端运行也可按 `Ctrl+C`。

程序默认不向局域网或公网开放。

## 2. 第一次运行

可选择三种方式：

1. **新建世界库**：填写世界库和第一个世界的名称，应用创建 Markdown 目录。
2. **打开现有目录**：选择或填写包含 `vault.yaml` 的目录。
3. **导入传输包**：选择 `.worldvault`，验证后创建一个新的世界库。

一个世界库可以包含多个世界。在“设置”中使用“创建另一个世界”。世界库与程序安装目录相互独立，替换程序不会删除正文。

## 3. 创建和编辑条目

点击“新建条目”，选择类型和状态。必填项只有标题；建议先写一段最确定的设定，再补充结构化字段。

内容状态：

| 状态 | 用途 |
| --- | --- |
| 正史 | 作者确认的当前事实，参与全部一致性检查 |
| 草稿 | 尚未采用的候选设定 |
| 传闻 | 世界内角色或组织的说法 |
| 废弃 | 保留历史但不再有效的设定 |

正文支持 Markdown。使用 `[[雾港]]` 链接标题或别名，使用 `[[loc_abcd1234abcd|雾港]]` 锁定稳定 ID。修改标题不会破坏 ID 链接。

“关系”用于强语义，例如：

- `member_of`：属于某组织；
- `located_in`：位于某地点；
- `causes`：导致某事件；
- `precedes`：发生在某事件之前；
- `parent_of`、`ally_of`、`enemy_of` 等。

关系目标填写目标条目的稳定 ID。普通提及不要伪装成强关系，直接写 Wiki 链接即可。

## 4. 时间与分支

“时间显示”保存世界内说法，例如“星历 312 年霜月下旬”；最早和最晚序号用于排序与模糊范围。如果只有大约时间，可填写一个范围，不要伪造精确日期。

时间线分支默认是 `main`。可为平行方案填写其他稳定名称。当前版本按分支标注和筛选内容，不自动把一个分支覆盖到另一个分支。

## 5. 图片、地图和附件

编辑条目时选择附件并点击“上传并插入正文”。应用接受常用图片、PDF、音频、TXT 和 CSV，单文件不能超过 50 MiB。附件保存在当前世界的 `assets/`，通过相对 Markdown 路径引用，并随 `.worldvault` 一起迁移。

不接受 HTML、脚本和可执行文件。Markdown 中的原始 HTML不会执行。

## 6. 搜索、时间线与检查

顶部搜索框检索标题、别名、正文和标签。“时间线”按内部序号排列有时间信息的条目。

“一致性检查”目前报告：

- 断开的 Wiki 链接；
- 同名或重复别名；
- 没有关联的孤立条目；
- 正史引用废弃条目；
- 关系目标不存在；
- 原因或前序事件确定晚于结果事件；
- 同一条目的 `located_in` 时间范围重叠。

检查只提供证据，不自动修改正文。

## 7. 导出与跨设备迁移

进入“迁移与审阅”。

### 世界传输包

“下载 `.worldvault`”导出可继续编辑的完整世界库。包中包含 Markdown、YAML、地图和附件，以及格式清单和逐文件 SHA-256；不包含 SQLite、日志、密钥和 Git 元数据。

在另一台设备安装相同或更新版本的程序，首次运行选择“导入传输包”。向已有世界库合并时，应用分别列出：

- 新增条目；
- 内容完全相同的条目；
- 同 ID 但正文不同的条目；
- 同路径但内容不同的附件或世界配置。

冲突默认保留本地。可以采用导入版本，或把冲突条目另存为草稿副本。提交合并前自动生成恢复快照。

### 静态审阅包

“下载审阅包”生成 HTML ZIP。解压后打开 `index.html`，无需安装程序即可检索和阅读。它是只读快照，不能替代 `.worldvault` 或历史备份。

## 8. 恢复与故障处理

- 索引结果不对：在“设置”或“一致性检查”点击重建索引。
- 程序无法启动：世界库仍是普通 Markdown，可用文本编辑器读取。
- 导入失败：原世界库保持不变；根据错误重新导出或修复磁盘空间。
- 合并结果不符合预期：恢复快照位于应用运行数据的 `runtime/backups/`。
- 程序升级失败：恢复旧程序目录并继续打开同一个世界库。
- 浏览器显示服务不可用：确认本地程序仍在运行，而不是只保留了浏览器标签页。

不要把静态审阅包作为唯一备份，也不要让多台设备同时写同一网络共享世界库。
