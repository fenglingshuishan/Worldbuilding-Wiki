# Changelog

## 0.4.0 - 2026-07-18

- 重做创作首页、分组导航与视觉系统，减少说明文字和常用操作层级。
- 增加地图主视觉、卡片入场、关系流动、地图标记和加载反馈动效，并支持系统“减少动态效果”设置。
- 内置“潮汐档案 · 示例全套数据”：13 条跨类型设定、自定义模板、时间、关系、地图图层和地点标记。
- 新建世界库默认载入示例；首页可一键删除或还原，删除只匹配固定示例标记，不会移除用户自建内容。
- 将示例地图和数据纳入 wheel、sdist 与冻结程序的资源校验。

## 0.3.0 - 2026-07-18

- 增加条目自动版本历史、统一差异视图和带乐观锁的一键恢复。
- 每条保留最近 100 个本地 Markdown 快照；恢复前再次保存当前版本。
- 明确 `.worldvault` 迁移当前内容、不包含隐藏本地历史的边界。
- 增加地图工作区、独立图层和引用地点条目的归一化坐标标记。

## 0.2.0 - 2026-07-17

- 将首页升级为创作驾驶舱，增加内容成熟度、构成、活动和关注队列。
- 增加可批量调整状态与标签的内容库工作台。
- 增加内置与世界库自定义模板中心，自定义模板可随传输包迁移。
- 增加全库关系网络，同时呈现结构化语义关系和 Wiki 提及。
- 将导航和视觉体系重构为八个稳定平台工作区。

## 0.1.1 - 2026-07-16

- 修复 GitHub Actions 在无 checkout 的发布作业中无法推断仓库的问题。
- 升级 GitHub 附件下载 Action，消除弃用运行时警告。
- 应用功能与世界库格式保持不变。

## 0.1.0 - 2026-07-16

- 建立本地优先的 Markdown 世界库。
- 提供条目编辑、搜索、关系、时间线和一致性检查。
- 提供 `.worldvault` 导入导出及静态 HTML 审阅包。
- 提供本机 Web 界面、Python 包和独立程序构建入口。
- 支持多世界、图片/地图/附件、页面内安全退出和导入恢复快照。
- 配置 Windows/Linux 原生发行构建与校验流程。
