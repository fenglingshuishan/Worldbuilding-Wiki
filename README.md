# Worldbuilding Wiki

面向个人创作者的本地优先世界观创作平台。当前版本 `0.4.0` 提供更紧凑的创作界面、可删除/还原的完整示例世界，以及版本恢复、地图、自定义字段、分支合并和可选 AI 提案工作流。

程序默认仅监听 `127.0.0.1`。世界观正文保存在用户选择的世界库中，SQLite 只保存可删除重建的索引。程序附带明确标为“示例全套数据”的虚构展示世界，不包含任何用户数据。

## 快速使用

普通用户应下载与操作系统匹配的独立发行压缩包，解压后运行 `WorldbuildingWiki.exe`（Windows）或 `WorldbuildingWiki`（Linux）。不需要安装 Python、Node.js 或数据库。

源码开发方式（工作目录为本仓库）：

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -e '.[dev]'
.venv/bin/worldbuilding-wiki
```

程序会打开本机浏览器。首次运行可新建世界库、打开已有 `vault.yaml`，或导入 `.worldvault`。新建时默认载入 13 条示例设定、关系、时间线和地图；首页可一键删除或还原。关闭浏览器不会停止后端；请在“设置”中选择“退出本地程序”。

### 本机 Windows / WSL 入口

Windows 桌面的 `WSL Workspace.lnk` 打开后，可通过 `05 Worldbuilding Wiki Project` 浏览唯一项目目录；`00 WSL Apps` 中提供 `06 Worldbuilding Wiki` 和 `06 Stop Worldbuilding Wiki`。启动入口使用项目内 `.venv`，仅监听 `127.0.0.1:3764`，运行日志写入被 Git 忽略的 `runtime/windows-launcher.log`。关闭浏览器不等于停止服务，应使用应用设置中的退出操作或配套停止入口。

## 已实现能力

- 人物、地点、组织、事件、群体、文化、规则、物件、概念和来源条目。
- 正史、草稿、传闻和废弃状态；别名、标签和时间线分支。
- `[[条目标题]]`、`[[条目 ID|显示名]]`、反向链接与局部关系。
- 多世界、模糊时间范围、语义关系和关系目标校验。
- 世界库的新建、打开、关闭，以及受名称确认保护的永久删除。
- 明确标记的“潮汐档案”完整示例世界；可一键删除、重置，且不触碰用户后来创建的内容。
- 紧凑创作首页、分组导航、快捷入口、地图主视觉和尊重“减少动态效果”偏好的微动效。
- 汇总规模、成熟度、创作队列、内容构成、热门标签和最近活动的创作驾驶舱。
- 支持多条件筛选、全选、批量调整状态和追加标签的内容运营工作台。
- 六套内置创作模板，以及保存在世界库中、可随传输包迁移的自定义模板。
- 五种自定义字段、不可变模板旧版本，以及先预览字段映射的跨模板迁移。
- 同时呈现结构化语义关系和正文 Wiki 提及的全库关系网络。
- 条目级自动版本历史、可视化统一差异，以及带并发保护的一键恢复。
- 中文标题、别名、正文和标签搜索。
- 图片、地图、PDF、音频和文本附件；单文件上限 50 MiB。
- 独立地图工作区、可见性图层和引用地点条目的归一化坐标标记。
- 分支谱系变体、继承/覆盖比较和逐条选择的可解释合并。
- 默认关闭、发送范围可见且绝不自动写正文的本地或外部 AI 提案层。
- 断链、重复别名、孤立条目、废弃引用、事件顺序和地点重叠检查。
- 带格式版本和 SHA-256 的 `.worldvault`；安全预检、冲突选择、暂存、恢复快照和失败回滚。
- 无需安装应用即可浏览的静态 HTML 审阅包。
- wheel/sdist、Linux 独立目录程序，以及 Windows/Linux 原生构建流水线。

```text

```

## 数据边界

- 世界库：Markdown、YAML、地图、附件和本地历史快照，是用户长期数据。
- 运行目录：SQLite 索引、导入暂存、临时导出和恢复快照。
- 应用目录：只读程序、依赖、前端和内置说明。
- 内置示例：应用只读资源中的虚构标准副本；选择载入后复制到世界库并随世界库迁移。
- 密钥：AI 默认关闭；启用时只从环境变量读取，不写入世界库、提案或传输包。

不支持多台设备同时写入同一个网络共享目录。跨设备编辑请显式导出、复制并导入 `.worldvault`，或在应用关闭后使用用户自行管理的 Git 工作流。

条目历史保存在世界库的隐藏 `.history/` 目录，最多保留每条最近 100 个版本。`.worldvault` 是当前内容的迁移快照，不包含该本地历史目录；需要保留完整编辑轨迹时应备份整个世界库目录。
